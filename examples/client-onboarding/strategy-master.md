## 1. Engagement Scope

**Client:** Meridian Financial Services (private, ~400 employees)  
**Engagement:** Operating Model Assessment + Onboarding Roadmap  
**Duration:** 8 weeks (2026-02-03 → 2026-03-27)  
**Budget:** Fixed fee — $38,000 USD

**In scope:**
- Assessment of current client onboarding process (discovery + interviews)
- Gap analysis against industry benchmarks
- 3-option roadmap for onboarding improvement
- Final presentation to leadership team (C-suite)
- One round of revisions on all deliverables

**Out of scope (explicit):**
- Technology implementation or vendor selection
- HR changes or org redesign
- Ongoing advisory retainer (separate contract if required)
- Any work related to Meridian's international operations

**Client-facing deliverables:**
1. Discovery Report (Week 3)
2. Gap Analysis (Week 5)
3. Options Roadmap (Week 7)
4. Final Presentation (Week 8)

## 2. Client Management Principles

### Principle 1: One source of truth per commitment
Any commitment made to the client (deadline, deliverable scope, meeting time)
is logged in the Canonical Numbers file. If it isn't there, it isn't a commitment.
AI must never invent or improvise commitments.

### Principle 2: Scope questions escalate before they expand
If any session produces work that appears to go beyond the engagement scope,
flag it explicitly before proceeding. Never expand scope in a document without
a written change order.

### Principle 3: Confidentiality firewall
All Meridian client data (internal process descriptions, interview content,
financials) is confidential. AI must not reference other clients, must not
draw comparisons using other client data, and must treat all Meridian material
as confidential by default.

## 3. Engagement Boundaries

| Boundary | Rule |
|----------|------|
| Scope | No deliverables outside Section 1 scope without signed change order |
| Technology | Do not recommend specific vendors by name in deliverables — use capability categories |
| Financials | Client's financial data used in analysis only; not in slide decks unless client approves |
| Timelines | All dates in canonical-numbers.md are final unless client agrees in writing |
| AI disclosure | If client asks whether AI tools were used, answer honestly — yes, for drafting and analysis support |

---

## 4. Communication Standards

**Client-facing tone:** Professional, warm, direct. Avoid jargon. Prefer clear
language over impressive vocabulary.

**Internal tone (running document, notes):** Candid and functional.

**Escalation:** If client raises anything outside scope, respond with:
"That's an interesting area — let me confirm whether it falls within our current
engagement scope and come back to you."
