# Example: Client Onboarding

**Domain:** Professional services / consulting  
**Use case:** Boutique strategy consultant onboarding a new enterprise client with AI collaboration

This example shows how a consultant uses LC-OS governance to manage a structured
client onboarding engagement, keeping client context, deliverables, and commitments
clean across dozens of AI sessions.

## What's in this folder

| File | Purpose |
|------|---------|
| `strategy-master.md` | Engagement scope, boundaries, client commitments |
| `running-document.md` | Current engagement phase, open items, session context |
| `canonical-numbers.md` | Fees, deliverable dates, client data used in work |
| `failure-log.md` | Errors and near-misses during the engagement |

## Why client work is high-stakes for AI collaboration

- Client confidentiality: AI must not mix context from different clients
- Commitment accuracy: What you tell a client you'll deliver must match what you actually deliver
- Scope creep: AI will helpfully suggest expanding scope without flagging the cost implication
- Tone: Client-facing writing has a different register than internal work

LC-OS governs all four. The failure log shows real cases.
